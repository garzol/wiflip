System settings

Purpose

This dialog box allows you to alter the persistant data of the board. It is accessible from the menu Tools>System settings. You must be connected through wifi (green connection led) before accessing data.

Using the System settings dialog box

Data is transferred instantly to the non volatile memory of the board, as soon as a checkbox is ticked/unticked, or a new startup mode is selected. Nevertheless, the data is not actually applied until a board reset is done (either by the physical reset switch or with the soft reset button of the WiFlip app)

Group box: Startup mode

There are 3 radio buttons:

  • Normal: This is the exact way the original pinball starts.
  • Miniprinter: This mode is an emulation of the original miniprinteer equipment from Recel.
  • Factory: Applies a safe, factory set of data.

Normal

The board will start with data from the last session (game settings, user data, etc...). This is the exact way the original pinball starts.

Miniprinter

This mode is an emulation of the original miniprinteer equipment from Recel. While the board is 100% compatible with the original miniprinter, it is highly dubious that you have one available at hand. This is why this mode was created. The board first load the miniprinter data, which is included in its dataset. Miniprinter data can be read/modified, saved/loaded from the memory inspection interface. Initially there is a "factory settings miniprinter" which is meant to be compatible with anay game. Miniprinter data will overwrite all the values that were previously stored by the pinball during the last session (handicaps, freeplays, model number... all data). This very same miniprinter data will be used at every startup of the machine until you select another mode.

This mode is not very useful, since you may prefer the game settings dialog box, which allows you to do almost the same things, in a more user-friendly evironment.

Factory

In this mode, factory settings are applied. To be used on first power on of the mother board to set up its presets or when things went wrong, or when you are lost. During next reset, this radio button will automatically switch to Normal. You will stay in factory mode, until you change it for Normal, or Miniprinter.

When factory radio button is selected, option checkboxes are dimmed. You cannot change them.

What is the factory settings state? When factory settings are applied, the options are all unticked, meaning that they are in the following state:

Options:

  • Coin rejector is active
  • coil self test done, potentially preventing the pinball to complete its test phase.
  • Standalone switch matrix is off. The MPU won't read switch states if not connected to a good mux board.
  • Unprotect coil channels. The complementary coil protection is activated.

Miniprinter data: This set of data is restored in such a state that it is compatible with any selected game.

RAM data: This set of data is restored in such a state that it is compatible with any selected game.

Production number: This field is restored to the real ID of the MPU (actually, it is a subset of the daughter card's serial

Model number: This is the current game installed (Given by Recel in the old times, 4-digit). Then, since factory settings remove any currently installed game, this number is reset to the actual factory installed game. CACE means "Crazy race".

Previous installed game and BIOS version are masked, after a factory reset. This means that the game applied will be the default one. The model number in the game settings indicates what are the factory default games and BIOS. CACE means "Crazy race" is the active game.

Group box: Option settings

Options are actually applied after a reset.

Options are:

  • Coin rejector is active
  • Skip coils self test
  • Standalone switch matrix
  • Unprotect coil channels

Coin rejector off

The coin rejector coil is located behind the coin door. It is rarely useful for home usage of a pinball. Besides, it makes annoying noise sometimes, and it consumes power for nothing. You can turn this coil off with this option.

Skip coils self test

At boot up, Recel pinballs perform a selftest. In several cases, the machine halts if a test did not pass. This is especially the case with coil test against short circuit or if play signal can't be detected. You may want to let the pinball continue its tests and not stop there, although it is generally not recommended. There is a specific case where this option, is useful: When the board is not connected to the pin (bench tests)

Standalone switch matrix

The state of switches (from the switch matrix) is read by the MPU board through the mux board. If the mux board is not connected to the MPU board (bench test, or standalone mode of the MPU board), then all swithches are detected in the closed state. Then, it is impossible to use the emulated switch matrix of the WiFlip app. This option was designed for preventing this situation, by reversing the state of switches by software.

Of course, this option must not be used in normal operation, since in such a case all switches will be seen closed by the MPU.

⚠When you can't start a game for example, this is a typical case. Always check that this option is set in accordance with your environment (standalone or embedded).

Unprotect coil channels

A special protection circuitry is provided for coil channels (C#/6..C#/F)

When the protection is activated, no coil may be kept activated for more than 3 seconds. If such a situation appears, the coil is turned off for 3 seconds, then returns in its state (from pinball software), for no more than 3secs if this state was on, etc...

This protection is not provided on the original Recel pins. Instead, there is a rearmable fuse of 3A located in the cabinet on the power supply board. This protection is useful, but may be insufficient in certain cases to protect power transistors or coils themselves. We have had the case sometimes, in particular with the ball rejector coil.

We thought it might be even more useful since we provided a way to emulate a permanent switch closing (triple clicking a given switch). This feature is provided to allows you to emulate hitting a target that is only reset by a bank reset. But if you triple click the Ball Home switch, then the pinball will trigger the ball home rejector permanently, and this is not good.

Conversely, you may find games were a coil needs to be permanently powered. We never met with such a situation, but you never know.

Moreover, if you want to have an exact replica of the original board, then you will tick this option

It is reecommended to leave this option unticked, to get the benefits of this complementary protection

Reset tool button

Click this button to trigger a pinball reset. All operations will stop immediatly, and a reboot occurs.

Click this button to see the applied changes. It is equivalent to click this button, press the physical on board button, or turn the power off, then on.
